# Typography

## Type scale

The typography scale uses a predetermined range of sizes for desktop and mobile.

A <LinkTo kind='components-text--text'>Text</LinkTo> component is provided to enable
a simpler api for applying text styles. So using the `Text` component rather than
hardcoding sizes in `css` is preferred.

If you ever have to apply a font size to css, they are exported implicity from the
`src/theme` under `fontSizes`, and can be accessed like so:

```js
import { fontSizes } from 'src/theme'

fontSizes.desktop.s1 // 12px
```

The font family used is the operating system font, so will look slightly different depending on the
person's setup. Still, this is preferred as it will look more _native_ and app-like.

Available font sizes are organized into `small (s)`, `medium (m)`, and `large (l)` groups.
Line height values are supplied as props to the <LinkTo kind='components-text--text'>Text</LinkTo>
component, so please head there for a full listing of accepted props.

Below you can see the mapping, as well as the rendered height of the text block depending on its
line height.
